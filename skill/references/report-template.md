# Review 报告模板

按以下结构输出报告。问题按 P0 → P1 → P2 排列，同级内按影响大小排序。没有问题的级别可省略。

```markdown
# Skill Review 报告 — <skill 名称>

- 审查日期：<YYYY-MM-DD>
- 审查范围：<全量 / 用户指定范围>，共 <N> 个文件
- 被审 skill 版本：<版本号 / 无版本标记>
- 审查方式：<主会话直接审查 / 拆分 N 个分支并行审查>

## 总体评价

<一段话：skill 的成熟度、最突出的问题类别、是否可用。>

## P0 问题（必须修复）

### [P0-1] <一句话标题>（检查项 <编号>）
- 位置：<文件路径:行号>
- 问题：<具体描述>
- 建议：<具体修改方案>

## P1 问题（应当修复）

### [P1-1] ...（同上结构）

## P2 建议（用户裁量）

### [P2-1] ...（同上结构）

## 需用户判决的问题

<集中列出 2.6 / 2.7 / 2.10 / 2.11 暂存的问题，随后用 AskUserQuestion 逐类询问。>

## 通过的检查项

<一行列出通过的检查项编号，如：1.1–1.3、2.1、2.3、2.5 通过。让用户知道哪些方面已确认没问题。>

## 不适用的检查项

<一行列出不适用的检查项编号及一句原因，如：3.1–3.4 不适用（纯文档 skill，无安装依赖）；2.12 不适用（非构建对话）。不要把不适用并入"通过"。>
```

## 条目示例

### [P0-2] 引用的转换脚本不存在（检查项 2.2）
- 位置：SKILL.md:42
- 问题：第 3 步要求运行 `scripts/convert.py`，但 `scripts/` 目录下无此文件，主流程在此断链。
- 建议：补充该脚本，或将步骤改为等价的 pandoc 命令并在文中给出完整命令行。

### [P1-3] 输出格式说明前后矛盾（检查项 2.3）
- 位置：SKILL.md:18 与 references/output.md:5
- 问题：SKILL.md 称默认输出为 JSON，references/output.md 称默认输出为 YAML。
- 建议：以脚本实际行为（源码第 30 行默认 `--format json`）为准，修正 references/output.md。
